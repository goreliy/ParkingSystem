# Решение проблемы загрузки YOLOv11 моделей

## Проблема

При попытке активировать модель YOLOv11 возникает ошибка:
```
AttributeError: Can't get attribute 'C3k2' on <module 'ultralytics.nn.modules.block' ...>
```

## Причина

Модели YOLOv11 требуют версию `ultralytics` 8.3.0 или выше. Текущая установленная версия (8.1.0) не поддерживает эти модели.

## Решение

### Шаг 1: Обновите ultralytics

Выполните в терминале (в виртуальном окружении проекта):

```bash
pip install --upgrade ultralytics>=8.3.0
```

Или если используете `requirements.txt`:

```bash
pip install -r requirements.txt --upgrade
```

(Файл `requirements.txt` уже обновлён до `ultralytics>=8.3.0`)

### Шаг 2: Перезапустите приложение

**ВАЖНО**: После обновления ultralytics обязательно перезапустите приложение, чтобы изменения вступили в силу.

### Шаг 3: Активируйте модель YOLOv11

После перезапуска попробуйте снова активировать модель YOLOv11 через интерфейс конфигурации.

## Проверка версии

Чтобы проверить установленную версию ultralytics:

```bash
pip show ultralytics
```

Должна быть версия 8.3.0 или выше.

## Дополнительная информация

- YOLOv11 был добавлен в ultralytics начиная с версии 8.3.0
- Старые версии (8.1.0 и ниже) не содержат необходимые классы архитектуры (`C3k2` и другие)
- Обновление ultralytics безопасно и не повлияет на работу YOLOv8 моделей

## Если проблема сохраняется

1. Убедитесь, что обновление прошло успешно: `pip show ultralytics`
2. Проверьте, что вы используете правильное виртуальное окружение
3. Попробуйте переустановить ultralytics: `pip uninstall ultralytics && pip install ultralytics>=8.3.0`
4. Проверьте логи приложения на наличие других ошибок
